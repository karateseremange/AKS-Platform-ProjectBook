# DOC-001 — Qualification de la documentation opérationnelle

| Élément | Valeur |
|----------|--------|
| **Type** | Document normatif |
| **Domaine** | Documentation opérationnelle |
| **Public** | Administrateurs, mainteneurs et contributeurs |
| **Statut** | Validé |
| **Version documentaire** | 1.0 |
| **Applicable à partir de** | AKS Platform v1.0.0 |

---

# 1. Objet

Le présent document définit les règles de qualification des guides opérationnels d'AKS Platform.

Il constitue la référence permettant d'identifier :

- les guides officiellement applicables ;
- leur périmètre ;
- leur état de validation ;
- les règles de maintenance documentaire.

Les guides référencés dans ce document sont les seuls documents opérationnels faisant officiellement partie d'une version publiée d'AKS Platform.

---

# 2. Périmètre

Le présent document couvre exclusivement les guides opérationnels de la plateforme.

Il ne décrit pas :

- l'architecture logicielle ;
- les décisions d'architecture (ADR) ;
- les décisions de gouvernance (DG) ;
- les standards de développement ;
- les notes de version.

Ces éléments sont documentés dans leurs documents de référence respectifs.

---

# 3. Principes de qualification

Avant toute publication officielle, chaque guide opérationnel doit répondre aux exigences suivantes :

- être rédigé au format Markdown ;
- avoir fait l'objet d'une relecture documentaire ;
- être cohérent avec la version publiée d'AKS Platform ;
- respecter les principes de confidentialité définis par la plateforme ;
- ne contenir aucune information sensible, secret ou donnée personnelle.

La validation d'un guide atteste qu'il peut être utilisé comme documentation officielle de la version concernée.

---

# 4. Base technique validée

La présente qualification est réalisée pour l'environnement suivant.

| Composant | Version |
|------------|---------|
| AKS Platform | 1.0.0 |
| AKS Platform Connector | 0.11.0 |
| WordPress | 7.0.1 |
| PHP | 8.3.23 |
| Google Apps Script | Runtime V8 |

Toute évolution de cette base technique devra faire l'objet d'une nouvelle qualification documentaire si elle impacte les guides concernés.

---

# 5. Guides opérationnels qualifiés

Les guides suivants font partie de la documentation officielle d'AKS Platform.

| Guide | Format Markdown | Format PDF | Public |
|--------|:---------------:|:----------:|--------|
| Guide d'installation | ✔ | ✔ | Administrateur |
| Guide d'administration | ✔ | ✔ | Administrateur |
| Guide utilisateur | ✔ | ✔ | Utilisateurs |
| Fiche d'information « Questionnaire Santé » | ✔ | ✔ | Représentants légaux |

Les fichiers Markdown constituent les documents de référence.

Les versions PDF sont générées exclusivement à partir des sources Markdown validées.

---

# 6. Contrôles réalisés

La qualification documentaire comprend notamment les contrôles suivants :

- cohérence des versions ;
- conformité avec la version publiée ;
- contrôle orthographique ;
- contrôle des liens internes ;
- contrôle des références documentaires ;
- vérification de la confidentialité des informations ;
- vérification de l'absence de données sensibles ;
- cohérence entre les versions Markdown et PDF.

---

# 7. Maintenance documentaire

Les règles suivantes s'appliquent à l'ensemble des guides opérationnels.

1. Les sources Markdown constituent la documentation de référence.

2. Les versions PDF sont générées uniquement après validation des documents Markdown.

3. Toute évolution d'un guide doit faire l'objet :

   - d'une mise à jour documentaire ;
   - d'une vérification de cohérence ;
   - d'une nouvelle qualification si les modifications impactent son contenu opérationnel.

4. Les noms des fichiers restent stables afin de garantir la pérennité des références documentaires.

---

# 8. Documents de référence

Le présent document s'appuie notamment sur :

- le README du Project Book ;
- les décisions d'architecture (ADR) ;
- les décisions de gouvernance (DG) ;
- les standards applicables ;
- les guides opérationnels ;
- les notes de version.

---

# 9. Conclusion

Les guides référencés dans le présent document sont qualifiés pour accompagner la publication officielle d'AKS Platform v1.0.0.

Ils constituent la documentation opérationnelle officielle de la plateforme.

Toute évolution future devra respecter les règles de qualification et de maintenance documentaire définies dans le présent document.
