# LOG-001
# Journalisation d'AKS Platform

Version : 1.0
Statut : Validé
Version du produit : V1.1

---

# 1. Objet

Ce document définit les principes de journalisation d'AKS Platform.

La journalisation permet d'assurer le suivi des traitements, le diagnostic des incidents, la traçabilité des opérations importantes et l'exploitation de la plateforme.

---

# 2. Objectifs

La journalisation doit permettre de :

- suivre les traitements exécutés ;
- faciliter le diagnostic des erreurs ;
- assurer la traçabilité des opérations importantes ;
- fournir des informations utiles à l'administration ;
- éviter la collecte de données inutiles.

---

# 3. Principes

Le système de journalisation doit être :

- centralisé ;
- cohérent entre les modules ;
- exploitable ;
- extensible ;
- sécurisé ;
- compatible avec les exigences de protection des données.

---

# 4. Événements journalisés

Les événements pouvant être enregistrés comprennent notamment :

- démarrage d'un traitement ;
- fin d'un traitement ;
- succès ;
- avertissement ;
- erreur ;
- modification importante d'un paramètre ;
- lancement d'une campagne ;
- opérations d'administration.

Les réponses détaillées du questionnaire santé ne doivent jamais être journalisées.

---

# 5. Niveaux

Les niveaux recommandés sont :

- Information ;
- Avertissement ;
- Erreur ;
- Critique.

Chaque événement doit utiliser le niveau le plus adapté.

---

# 6. Contenu minimal

Un journal doit pouvoir contenir :

- date et heure ;
- origine ;
- module concerné ;
- type d'événement ;
- niveau ;
- description ;
- résultat.

Les données personnelles ne doivent être enregistrées que lorsqu'elles sont strictement nécessaires.

---

# 7. Sécurité

La journalisation doit :

- protéger les informations sensibles ;
- masquer les secrets ;
- respecter les droits d'accès ;
- éviter toute fuite d'information.

---

# 8. Consultation

Le tableau de bord d'administration doit permettre :

- la consultation des journaux ;
- le filtrage ;
- la recherche ;
- l'affichage des erreurs récentes.

---

# 9. Conservation

La durée de conservation des journaux est définie par les besoins d'exploitation et les contraintes réglementaires.

Les journaux devenus inutiles doivent pouvoir être archivés ou supprimés selon la politique retenue.

---

# 10. Critères d'acceptation

Le système est conforme lorsque :

- les événements importants sont journalisés ;
- les erreurs sont identifiables ;
- les secrets ne sont jamais exposés ;
- les modules utilisent un mécanisme commun ;
- les journaux sont consultables depuis l'administration.

# 12. Conclusion

La journalisation constitue un service commun essentiel d'AKS Core. Elle garantit une exploitation fiable, une meilleure maintenance et prépare l'évolution de la plateforme sans compromettre la confidentialité des données.
