# Objet du document

Ce document définit les principes qui guident toutes les décisions du projet.

# Principes

## Documentation Before Code

Toute décision structurante est documentée avant son implémentation.

## Architecture Before Implementation

L'architecture précède le développement.

## Generic by Design

Les composants sont conçus pour être réutilisables lorsque cela est pertinent.

## YAGNI fonctionnel

Aucune fonctionnalité n'est développée sans besoin métier identifié.

## Simplicité

La solution la plus simple répondant au besoin est privilégiée.

## Maintenabilité

Chaque décision doit faciliter les évolutions futures.
