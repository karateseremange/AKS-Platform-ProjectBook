# 1. Objet

Le présent document définit le Design System d'AKS Platform.

Il constitue la référence commune pour la conception et l'implémentation de l'ensemble des interfaces utilisateur de la plateforme.

Son objectif est de garantir :

- une expérience utilisateur homogène ;
- une maintenance simplifiée ;
- une évolution maîtrisée des composants ;
- une accessibilité cohérente.

Le Design System ne décrit pas l'identité graphique du produit. Il définit les composants, les règles de composition et les principes de leur utilisation.

---

# 2. Principes

Tous les composants respectent les principes suivants :

- responsabilité unique ;
- interface publique stable ;
- indépendance vis-à-vis du métier ;
- réutilisabilité ;
- composition plutôt qu'héritage ;
- cohérence avec UX-001 ;
- *Accessibility by Design*.

---

# 3. Architecture des composants

Chaque composant suit le modèle conceptuel suivant :

```text
UI Component
├── Interface publique
├── Propriétés
├── États
├── Événements
└── Contenu
```

Règles :

- un composant expose uniquement son interface publique ;
- un composant ignore son contexte métier ;
- un composant ne modifie jamais directement un autre composant ;
- les composants sont conçus pour être composés entre eux.

---

# 4. Langage visuel

Le Design System définit un langage visuel commun basé sur les états fonctionnels de la plateforme.

Les états normalisés sont :

- Succès ;
- Information ;
- Attention ;
- Incident ;
- Désactivé.

Chaque état comporte systématiquement :

- une couleur ;
- une icône ;
- un libellé textuel.

Aucune information essentielle ne repose uniquement sur la couleur.

| Élément | Présence |

| Content | Obligatoire |

# 7. Typographie et mise en page

Le Design System définit :

- la hiérarchie des titres ;
- les styles de texte ;
- les indicateurs numériques ;
- les espacements ;
- la grille de mise en page ;
- les alignements.

Ces règles assurent une présentation homogène sur l'ensemble de la plateforme.

---

# 8. Responsive

Les composants sont conçus pour fonctionner sur :

- ordinateur ;
- tablette ;
- smartphone.

Le responsive adapte uniquement la disposition des éléments. Le comportement fonctionnel reste identique.

---

# 9. Accessibilité

Le Design System applique le principe suivant :

> **Accessibility by Design**

Les composants doivent notamment :

- être utilisables au clavier ;
- présenter un contraste suffisant ;
- fournir un libellé pour chaque état ;
- ne jamais transmettre une information uniquement par la couleur ;
- être compatibles avec les technologies d'assistance.

---

# 10. Gouvernance du Design System

Le Design System évolue progressivement.

Chaque composant appartient à un niveau de maturité.

| Niveau | Description |
|---|---|
| **Core** | Composant fondamental utilisé dans toute la plateforme. |
| **Standard** | Composant officiellement recommandé pour les modules. |
| **Extension** | Composant répondant à un besoin particulier mais conçu pour être partagé. |
| **Experimental** | Composant en cours de validation, sans garantie de stabilité. |

Évolution d'un composant :

```text
Experimental
        │
        ▼
Extension
        │
        ▼
Standard
        │
        ▼
Core
```

Un composant ne change de niveau de maturité qu'après avoir démontré son utilité sur plusieurs cas d'usage concrets.

Les modules ne créent pas de variantes locales des composants standards sans justification documentée.

Tout nouveau composant partagé est d'abord ajouté à UI-001 ou à une fiche de composant rattachée au présent contrat avant d'être généralisé à la plateforme.
